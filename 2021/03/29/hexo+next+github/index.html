Hexo + next + Github

新手搭站部署Github必知

Git pages

GitHub Pages is designed to host your personal, organization, or project pages from a GitHub repository.

You get one site per GitHub account and organization, and unlimited project sites. Let‘s get started.

Github Pages 识别仓库中的一个分支里的网站，并提供一个免费的域名，可以利用该域名访问你的网站。

域名问题

域名可以作为中转。

如：域名A，域名B，IP

B -> IP

A -> B

A -> B -> IP

搭建个人博客的简单路径

  • 选择博客框架 如Hexo
  • 部署至类似Github平台
  • 购买自己域名，绑定Github域名

环境准备

  • Ubuntu 16.04.12

  • Node.js 10.13.0 (需不低于10.13)

  • Git

  • npm 5.6.0 (Node.js & npm 使用 nvm install 10.13.0)

  • Hexo-cli

使用Hexo博客框架初始化个人博客

  • 创建文件夹

  • 初始化，生成文件

    • 修改文件配置文件

      https://hexo.io/zh-cn/docs/configuration 官方如下

      其他默认配置可以不修改

    • 下载主题，渲染博客框架

      比如Next，找到官方github中的next的教程

      注意：这里我使用了github镜像，加速克隆。并且指定下载到themes文件夹下

    • 修改_config.yml中的theme为next

    • 现在可以开启使用了

      此时，如果是在本机上安装，则可以使用localhost:4000查看网站

创建仓库

  • ⚠️仓库名Repository Name：[Your_username].github.io <username为github的用户名，如果自创，后期会有不兼容网页等问题，具体见官方文档Github Pages>

文件夹Git初始化

  • 在博客目录下（_config.yml所在目录）

将会生成.gitignore文件，会在后期部署至Github上上传指定文件夹

如果有public/，没有则补充

将Hexo一键部署到Github上

  • 下载依赖包

  • 修改配置文件_config.yml，最下方deploy

    repo：即为自己的仓库

  • 修改Gihub上的设置

    修改默认分支为master

    修改Github Pages的分支为master 保存

  • 回到目录/home/blog

    注意：在使用hexo d会让你提供账号和密码，hexo不会保存你的账号密码

此刻你就可以使用github pages 提供的域名，登陆你的网址，上图绿色部分
