进制学习 Class 1

1.1定义

N进制，逢N进1，使用位权表示法。

1.2常用进制

  • 二进制

    • 进制符号：0，1
    • 进制表示：xxxxB

  • 十进制

    • 进制符号：0-9
    • 进制表示：xxxxD

  • 十六进制

    • 进制符号：0-9，A-F
    • 进制表示：xxxxH

  • 进制转换

    • 十进制转其他进制

      方法：除基数取余法，自下至上排列

      例如：232转16进制，基数16，余数分别为8 E（14）即，十六进制为：E8H

    • 十进制小数转其他进制

      方法：乘基数取整数法，自上至下排列

      例如：0.8125转2进制，基数2，整数分别为1 1 0 1 即二进制为0.1101B

    • 其他进制转为十进制

      方法：展开求和

      例如：11111110B

    • 二进制、八进制、十六进制相互转换

      方法：3个二进制位表示1个八进制，4个二进制位表示1个十六进制

      例如：E8B = 1110 1000 B = E8B

1.3 机器数

机器数：符号数字化，数字在计算机中表示为二进制

机器数的原码：最高位为1个符号位、其余位表示数值。

机器数的反码：对于正数而言，原码==反码，对于负数而言，反码为符号位不变，其余位取反（0变1，1变0）。

机器数的补码：对于正数而言，原码==反码，对于负数而言，为其反码+1

为什么会有反码？

计算机中读取数字的形式不像一个数轴可以无限延伸，计算机中用二进制数表示数值，所以就会有以下**规则**。

正数中：原码==反码==补码

为什么负数中不这样？如果采用最高位为符号位的形式，会出现，2个0，且无法进行减法运算。

如果**采用下图规则**，即当前补码的规则：

  • **正数的补码等于原码，负数的补码等于反码+1**，便可以做到

假设 6 - 5 ：6的补码 - 5的补码 = 6的补码 + （-5）的 补码 = 1的补码

0110 - 0101 == 0110 + 1101 == 0001 （最高位自然流出）

即：补码的作用：规避了减法运算。

补码的运算

[x]指x的补码

[x] + [y] = [x+y]

[x] - [y] = [x] + [-y]

1.4 二进制的运算

与、或、非、异或

与：只有两者均为1，则值为1，其余为0

或：只有两者均为0，则值为0，其余为1

非：取反

异或：两者相同时，值为1，不同时为0‘

进位和借位

进位：加法时向前进1

借位：减法时向后借1

判断溢出的三种方式

溢出：运算结果超出所能表示的范围，则产生溢出

自然流出：运算结果不影响表示，即只要非一下三种溢出方式，则为自然流出

  • 双位符号法：在最高符号位前，增加1位，与符号位相同。如果最终运算结果两个符号位值不一样，则产生溢出。
  • 一般判别法：同号数相加，结果符号位不同则为溢出，异号数相加，运算结果与被减数的符号不同则产生溢出。
  • 进位位判别法：值的最高位向前进位，与符号位向前进位同时发生，不溢出，不同则为溢出。

1.5 编码

  • 二进制编码

    BCD（Binary-Coded Decimal）‎：是一种将十进制表示为二进制的编码形式。用4位二进制表示一个十进制位。

  • 英文字符编码

    ASCII码（American Standard Code for Information Interchange）：用1个字节表示一个字符，其中最高位为0，剩余7位表示字符，共128个字符。
